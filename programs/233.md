# Task #233 — TelosBoard: Mission Control UI for Agent Work

## Telos Description
Mission Control dashboard + agentic work harness. Two intertwined problems:

## Problem 1: Visibility (Dashboard)
Chat is terrible for state visibility. Andreas cant tell:
- Is Conductor stale, missed a notification, or actively cooking?
- Whats the current task state/performance?
- Whats next in the queue?
- Which context belongs to which task?

Solution: Web UI showing active workstreams as cards with:
- Task title + description
- Current status (idle / cooking / blocked / done)
- Live KPIs (R@5, MC_Acc, etc) that agents update
- Queue/backlog visibility
- Last activity timestamp
- CC process status (running PID, elapsed time, log tail)

## Problem 2: Work Continuity (Agentic Harness)
Currently using 30-min cron jobs to keep work flowing. Tasks should chain:
- Task A completes -> immediately triggers Task B
- No 30-min idle gap between tasks
- Self-scheduling work loop with break conditions
- Human can pause/steer/reprioritize from dashboard

Solution: Work loop harness that:
- Picks next task from queue
- Launches CC / subagent
- Monitors completion (event-driven, not polling)
- On completion: logs results, picks next task, continues
- Human override: pause, skip, reprioritize via dashboard
- Automatic notification on completion + state change

## Design Principles
- Dashboard reads from Telos DB + agent state files
- Agents update KPIs via simple CLI/API calls
- No new infra — static HTML + SQLite reads (like Telos viz)
- Work harness is an OpenClaw feature request or custom orchestrator script
