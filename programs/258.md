## How to run
Smoke test (1 conv, ~2 min):
```
/home/linuxbrew/.linuxbrew/bin/python3 src/pipeline.py --config configs/<your_config>.yaml --entity-bench --conversations 1
```

Full run (3 convs, ~5 min):
```
/home/linuxbrew/.linuxbrew/bin/python3 src/pipeline.py --config configs/<your_config>.yaml --entity-bench --conversations 3
```

## Metric extraction
After a run, find the latest entity-scores.json in results/:
```
cat results/<latest>-entity-scores.json | python3 -c "import json,sys; s=json.load(sys.stdin); n=len(s); print(f'R@5={sum(1 for e in s if e.get(\"is_hit\"))/n:.4f} MC={sum(1 for e in s if e.get(\"mc_correct\"))/n:.4f}')"
```

## Keep/discard rules
- KEEP if R@5 improves by >= 0.005 OR MC improves by >= 0.01
- KEEP if both metrics stay within 0.01 of previous AND code is simpler
- DISCARD otherwise (git revert to EXP-017 base: aa7d8b44 in mem90)
- Always KEEP if new record on either metric
- Always run smoke test first; abort full run if smoke R@5 < 0.70

## Design mandate
- Entity = any noteworthy thing (people, systems, pets, projects, places, orgs)
- Every change must be transferable to MemoryHub production
- No knowledge graphs — flat entity register
- Dual timestamps: effective_date (if stated) + reported_at (always)
- Evidence references mandatory
- Do NOT add synthetic chunks to the retrieval pool in bulk (confirmed failure 3x: EXP-012, EXP-019, relationship chunks)
- Do NOT expand reranker pool beyond 20 candidates (confirmed ceiling: EXP-018)
- Do NOT use chunk window > 1 (catastrophic: EXP-007)
